Implantação e automatização de operações de um comércio de bijuterias e acessórios por meio de software web

Projeto Integrador para os cursos de Engenharia da Computação, Ciências de Dados e Bacharelado em Tecnologia da Informação

RESUMO

Este relatório tem como objetivo apresentar o desenvolvimento do projeto integrador focado na criação de um software web com banco de dados para controle de estoque, visando facilitar e otimizar a organização e gestão do estoque de uma loja de bijuterias e acessórios. O comércio utilizava planilhas simplificadas que já não atendiam às necessidades do negócio, gerando dificuldades no controle de entrada e saída de produtos, compras desnecessárias e falta de visibilidade sobre o estoque disponível. O software foi desenvolvido com o foco principal em automatizar processos, reduzir erros e aumentar a eficiência na gestão do estoque, otimizando os processos de gerenciamento do negócio. Foi aplicada a metodologia de desenvolvimento web utilizando a tecnologia Python e o framework Django, praticando controle de versão através da plataforma GitHub e integração ao banco de dados com SQL. Adotando uma linguagem de desenvolvimento simples, mas com alta eficiência, e uma interface de fácil utilização, proporcionamos uma solução prática para os problemas de estoque encontrados na loja, tornando os processos diários mais ágeis.

PALAVRAS-CHAVE: controle de estoques; desenvolvimento web; automatização de processos; microempresas; banco de dados.

1. Introdução e Justificativa

A evolução tecnológica tem impactado significativamente o gerenciamento de negócios, tornando a utilização de ferramentas automatizadas essencial para a eficiência e rentabilidade. Observou-se que problemas de estoque e gerenciamento de produtos são comuns em diversos segmentos de comerciantes locais.

Para este trabalho, a empresa Lamavie, do ramo de acessórios e bijuterias, relatou dificuldades na gestão de suas operações, principalmente no controle de estoque. O uso de métodos manuais resultava em compras excessivas e desnecessárias, aumentando os custos e impactando os recursos financeiros da empresa. Diante dessa problemática, o projeto propõe a criação de um software web acessível e de fácil manejo para solucionar essa deficiência. A solução foi desenvolvida utilizando tecnologias como Python, Django, SQL e GitHub, baseando-se nos conceitos adquiridos nas disciplinas do curso.

2. Objetivos do Projeto

O propósito principal do projeto é desenvolver um sistema digital para que a empresa estudada tenha mais controle sobre suas operações, visando a otimização de tempo e recursos e contribuindo para seu crescimento.

Objetivos Específicos

  • Identificar as dificuldades nos processos atuais da empresa.
  • Analisar e propor soluções de software aplicáveis ao negócio.
  • Desenvolver um software web com banco de dados para controle de estoque.
  • Automatizar processos para reduzir erros e aumentar a eficiência na gestão.

3. Metodologia

Para a elaboração do projeto, foi adotada uma metodologia para sistemas web integrada, segregada em três etapas principais: interpretar, criar e aplicar. O objetivo foi compreender as atividades e necessidades reais do comércio para que a solução fosse eficaz e de fácil utilização.

A coleta de informações envolveu entrevistas, formulários e observação direta com os proprietários da empresa. A partir da análise qualitativa dos dados, foram identificados os principais problemas, como controle de estoque deficiente e falta de integração entre vendas e compras. O desenvolvimento seguiu uma abordagem iterativa, com a criação de um protótipo, coleta de feedback dos usuários e ajustes contínuos até a obtenção de um produto funcional e compatível com a realidade do negócio.

4. Desenvolvimento e Aplicação das Disciplinas

O projeto integrou conhecimentos de diversas disciplinas cursadas na UNIVESP, sendo fundamental para a transformação do conceito teórico em uma aplicação prática.

Conceitos de pensamento computacional foram aplicados para decompor o problema em partes menores e planejar as funcionalidades do sistema. A lógica de programação, variáveis, estruturas de decisão e repetição, vistas em Algoritmos e Programação, foram a base para a codificação do sistema.

A construção do sistema seguiu o modelo de Programação Orientada a Objetos. Foi utilizada a linguagem Python com o framework Django, além de tecnologias como HTML, CSS e Javascript. Conceitos como encapsulamento, herança e tratamento de exceções foram aplicados para criar as funcionalidades.

Na disciplina de Banco de Dados, foram criados os modelos lógico e físico usando MySQL. Conhecimentos de Infraestrutura para Sistemas de Software foram essenciais para entender o modelo cliente-servidor, protocolos e para testar a aplicação em servidores locais e em nuvem.

5. Alinhamento com o PPC

O desenvolvimento do software Unistock seguiu estritamente as diretrizes das Práticas como Componente Curricular (PCC) descritas no Projeto Pedagógico do Curso, em especial as disciplinas de Projeto Integrador.

Projeto Integrador em Computação I

  • Objetivo do PPC: Desenvolver um software com framework web que utilize noções de banco de dados, praticando controle de versão.
  • Ementa do PPC: Resolução de problemas; Levantamento de requisitos; Desenvolvimento web com framework; HTML; CSS; Banco de Dados; Controle de Versão.
  • Aplicação no Projeto: O grupo atendeu a todos os pontos ao identificar o problema de gestão de estoque, levantar os requisitos com a comerciante, utilizar o framework Django para o desenvolvimento web, integrar um banco de dados SQL e gerenciar o código com Git/GitHub.

Projeto Integrador em Computação II

  • Objetivo do PPC: Desenvolver um software com framework web que utilize banco de dados, inclua script web (Javascript), nuvem, uso de API, acessibilidade, controle de versão e testes.
  • Ementa do PPC: Resolução de problemas; Levantamento de requisitos; Desenvolvimento web com framework; HTML, CSS; linguagem de script; Banco de Dados; Controle de Versão; Nuvem; API; Acessibilidade; Testes; Análise de dados.
  • Aplicação no Projeto: A evolução do projeto contemplou a utilização de JavaScript para aprimorar a interface, a aplicação de conceitos de nuvem para hospedagem e a realização de testes para garantir a funcionalidade da aplicação, alinhando-se aos requisitos desta etapa.

6. Resultados: A Solução Final

Considerando o feedback dos usuários e o cronograma do projeto, o grupo desenvolveu uma solução final funcional com as funcionalidades essenciais para o controle das operações da loja. O sistema, nomeado Unistock, foi estruturado com foco em simplicidade, acessibilidade e eficiência.

Funcionalidades Implementadas:

  • Cadastro e Gerenciamento: Módulos para cadastrar e gerenciar produtos, clientes e fornecedores.
  • Painel de Controle (Dashboard): Interface visual com gráficos e indicadores sobre vendas, despesas e estoque.
  • Relatórios Gerenciais: Geração de relatórios sobre os produtos mais vendidos, controle de estoque e movimentação financeira, com opção de exportação para Excel e PDF.
  • Acesso Web: O sistema pode ser acessado via navegador, sem necessidade de instalação.

7. Considerações Finais

A aplicação do sistema demonstrou boa aceitação pela interface e utilidade das funções, especialmente no controle de estoque, trazendo um aspecto mais profissional ao negócio. O projeto, embora não seja uma solução inovadora, apresenta-se como uma ferramenta de baixo custo e adaptável à realidade de comércios de pequeno porte sem orçamento para grandes sistemas.

A experiência revelou a constante evolução dos requisitos como um desafio, reforçando que o desenvolvimento de software é um processo contínuo. Apesar de nem todas as funcionalidades previstas terem sido implementadas devido a limitações de tempo e nível técnico, acredita-se que a proposta da Univesp foi atendida, com a entrega de uma aplicação web funcional que contempla os requisitos técnicos e de aprendizado do projeto.
